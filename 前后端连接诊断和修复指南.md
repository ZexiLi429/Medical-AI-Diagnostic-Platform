# 前后端连接诊断和按钮修复指南

## 🔍 问题诊断

您遇到的问题：**前端SAM按钮点不了**

可能原因：
1. ❌ 按钮没有添加到工具栏
2. ❌ 按钮的evaluate条件不满足
3. ❌ 命令未正确注册
4. ❌ 后端未启动或端口不对
5. ❌ CORS跨域问题

## 🔧 解决方案

### 问题1: 按钮灰色（不可点击）

**原因**：没有创建分割对象，`evaluate.cornerstone.segmentation` 条件不满足

**解决方案A - 快速修复**：先创建一个分割对象

1. 打开OHIF，选择 **MedSAM Viewer** 模式
2. 加载DICOM数据
3. **在右侧面板点击 "Add Segment"** (添加分割)
4. 此时 `StoreOriginSlice` 和 `SAMApply` 按钮应该可以点击了

**解决方案B - 修改代码**（移除分割检查，让按钮始终可用）

修改文件：`modes/sam/src/toolbarButtons.ts`

```typescript

  uiType: 'ohif.toolBoxButton',

    icon: 'store-origin-slice',
    label: 'Store Original Slice',

    commands: 'storeOriginSlice',
    evaluate: [
      'evaluate.action',
      {
        name: 'evaluate.viewport.supported',
        unsupportedViewportTypes: ['video', 'wholeSlide'],
      },
      { name: 'evaluate.cornerstone.segmentation'},  // ❌ 移除这个
    ],
  },
},

// ✅ 修改为（不需要分割对象也可以使用）
{
  id: 'StoreOriginSlice',
  uiType: 'ohif.toolBoxButton',
  props: {
    icon: 'store-origin-slice',

},
```

### 问题3: CORS跨域错误

**症状**：浏览器Console显示 `Access to fetch at 'http://localhost:8000/segment' from origin 'http://localhost:3000' has been blocked by CORS policy`

**检查后端CORS配置**：`medsam_service.py` 第20-27行

```python
# 确保包含前端地址
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000", "http://localhost:8042"],  # ✅ 确认3000在列表中
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

**如果前端端口不是3000**（比如3001）：

```python
allow_origins=[
    "http://localhost:3000",
    "http://localhost:3001",  # ✅ 添加实际端口
    "http://localhost:8042"
],
```

## 🐛 故障排查

### 问题A: 修改后按钮还是灰色

**清理缓存重新构建**：

```powershell
cd miscada-project-master
rm -rf node_modules/.cache
yarn build
yarn run dev
```

或者强制刷新浏览器：`Ctrl + Shift + R`

### 问题B: 点击按钮没反应

**打开浏览器开发者工具**（F12）查看Console：

```javascript
// 1. 检查命令是否注册
window.commandsManager.getCommand('storeOriginSlice')
// 应返回: { commandFn: function, ... }

// 2. 手动执行命令测试
window.commandsManager.runCommand('storeOriginSlice')
// 应显示成功通知
```

### 问题C: 后端请求失败

**检查Network标签**（F12 → Network）：

1. 点击 SAM Apply 按钮
2. 查看是否有请求发送到 `http://localhost:8000/segment`
3. 查看状态码：
   - ✅ **200 OK** - 成功
   - ❌ **404 Not Found** - 路由不存在（检查后端是否启动）
   - ❌ **500 Internal Server Error** - 后端错误（检查模型文件是否存在）
   - ❌ **CORS error** - 跨域问题（检查CORS配置）

### 问题D: 模型文件缺失

**症状**：后端启动时报错 `FileNotFoundError: medsam_vit_b.pth`

**解决方案**：下载模型文件

```powershell
# 创建目录
cd MedSAM-main
mkdir -p work_dir/MedSAM

# 下载模型（约180MB）
# 方法1: 使用浏览器下载
# https://drive.google.com/drive/folders/1ETWmi4AiniJeWOt6HAsYgTjYv_fkgzoN

# 方法2: 使用gdown工具
pip install gdown
gdown "https://drive.google.com/uc?id=1UAmWL88roYR7wKlnApw5Bcuzf2iQgk6_" -O work_dir/MedSAM/medsam_vit_b.pth
```
